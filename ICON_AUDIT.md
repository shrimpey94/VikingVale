# Icon Audit — VikingVale

Generated by greping every item-id literal referenced in: `ItemPrices.gd`,
`ShopCatalog.gd`, `Fishing.gd`, `SeaMonsters.gd`, `Monster.gd`,
`Interactable.gd`, and every recipe table in `HUD.gd` (forge / cook / craft /
construction / rune-smithing / buildables). Then diffed against the existing
`assets/icons/*.png` set (724 icons currently shipping).

The 37 missing icons below cover every gap surfaced by the audit. The
ID-only filenames are what the runtime expects — when the `drop_<id>.png`
form is also missing the in-world LootDrop will fall back to a coloured
circle and lose the icon-on-ground polish.

## What this list does NOT cover

- **Bar redesigns**: the audit can only flag references that have no PNG at
  all. "Distinct icons rather than color swap" is a visual-art question
  the file system can't answer.
- **Drop variants** (`drop_<id>.png`): a separate sweep would list the same
  37 items with the `drop_` prefix; mirroring the base icon at smaller
  size is the standard pattern.
- **Per-tier construction items** (oak_campfire, pine_campfire, …):
  intentionally orphaned — campfire / storage_crate / torch_post / bookshelf
  moved to single-tier crafting recipes, so the tier-prefixed variants are
  no longer reachable through any UI. Leaving the existing PNGs in place is
  harmless (they render correctly if any player still has one in inventory).

## Generation note

I do not have an image-authoring tool available in this environment, so I
cannot produce the missing PNG files. The list above is the actionable
deliverable — point an artist / image-gen pipeline at it, drop the
results in `assets/icons/`, and Godot's auto-import will register them
on the next editor focus.
